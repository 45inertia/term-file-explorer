## Notes

Documentation beyond diagrams to reach a professional level:
- README: problem statement, screenshot/GIF, build instructions for linux and windows, architecture
  diagram embedded at the top
- ADRs(Architecture Decision Records): short markdown files, one per significant decision. This
  separates code that works from code whose reasoning is legible to someone else. A short template
  (Context/Decision/Consequences) is enough
- CONTRIBUTING.md: useful for documenting build/test conventions so they aren't reinvented every
  session
